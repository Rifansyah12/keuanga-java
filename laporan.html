BAB I
Pendahuluan

1.1 Latar Belakang

Pengelolaan keuangan pribadi merupakan hal yang sangat penting dalam kehidupan sehari-hari. Banyak individu kesulitan dalam memantau pemasukan dan pengeluaran mereka secara teratur karena tidak adanya sistem yang terorganisir. Kebiasaan mencatat keuangan secara manual dinilai kurang efisien dan rentan terhadap kesalahan serta kehilangan data.

Perkembangan teknologi informasi, khususnya di bidang pemrograman berbasis web, membuka peluang untuk membangun sistem catatan keuangan yang mudah digunakan, terstruktur, dan dapat diakses kapan saja melalui browser. Dengan memanfaatkan teknologi Java khususnya Spring Boot sebagai back-end dan antarmuka web sebagai front-end, sebuah aplikasi catatan keuangan pribadi dapat dikembangkan dengan efisien.

Dalam rangka memenuhi tugas Ujian Akhir Semester mata kuliah Pemrograman Java Lanjut, penulis mengembangkan sebuah aplikasi bernama CatatKeu — Catatan Keuangan Pribadi berbasis web. Aplikasi ini mengimplementasikan seluruh konsep yang telah dipelajari selama perkuliahan, mulai dari OOP, Collection Framework, Lambda Expression, Stream API, RESTful API, Spring Boot, JPA Repository, hingga integrasi front-end dan back-end.

1.2 Tujuan

Tujuan dari pengembangan aplikasi ini adalah:

  1. Membangun aplikasi web catatan keuangan pribadi yang fungsional dan dapat digunakan secara nyata.
  2. Mengimplementasikan konsep Object Oriented Programming (OOP) dalam perancangan entitas sistem.
  3. Menerapkan Collection Framework (List, Set, Map) dalam pengelolaan data.
  4. Menggunakan Lambda Expression dan Stream API untuk pemrosesan dan analisis data keuangan.
  5. Membangun RESTful API dengan Spring Boot sebagai back-end server.
  6. Memanfaatkan Spring Data JPA sebagai layer akses database MySQL.
  7. Mengintegrasikan front-end HTML/CSS/JavaScript dengan back-end Spring Boot.

1.3 Ruang Lingkup

Ruang lingkup aplikasi CatatKeu meliputi:

  • Manajemen Kategori — Pengelolaan kategori transaksi (pemasukan dan pengeluaran).
  • Manajemen Transaksi — Pencatatan pemasukan dan pengeluaran harian dengan filter per bulan.
  • Manajemen Budget — Pengaturan batas pengeluaran per bulan beserta visualisasi penggunaan.
  • Dashboard — Ringkasan keuangan bulanan dengan grafik interaktif (Chart.js).
  • Login — Sistem autentikasi sederhana berbasis username dan password.
BAB II
Landasan Teori

2.1 Object Oriented Programming (OOP)

Object Oriented Programming (OOP) adalah paradigma pemrograman yang mengorganisir kode ke dalam objek-objek yang merepresentasikan entitas dunia nyata. OOP memiliki empat pilar utama yaitu Encapsulation, Inheritance, Polymorphism, dan Abstraction.

Dalam aplikasi ini, OOP diterapkan melalui pemisahan lapisan yang jelas antara Entity (model data), Repository (akses data), Service (logika bisnis), dan Controller (REST endpoint). Setiap entitas seperti Kategori, Transaksi, dan Budget direpresentasikan sebagai class Java dengan atribut dan method yang terenkapsulasi.

2.2 Collection Framework

Java Collection Framework menyediakan sekumpulan antarmuka dan kelas untuk menyimpan dan memanipulasi kelompok objek. Tiga struktur data utama yang digunakan dalam aplikasi ini adalah:

  • List — Struktur data berurutan yang memungkinkan duplikasi. Digunakan untuk menyimpan daftar transaksi, kategori, dan budget dari database.
  • Map — Struktur key-value pair. Digunakan untuk mengelompokkan total pengeluaran per kategori (Map<String, Double>) dan ringkasan bulanan.
  • Set — Struktur data tanpa duplikasi. Digunakan untuk mengambil kumpulan tahun unik dari seluruh transaksi.

2.3 Lambda Expression

Lambda Expression merupakan fitur yang diperkenalkan pada Java 8 yang memungkinkan penulisan fungsi anonim secara ringkas. Lambda mempermudah penulisan kode yang sebelumnya memerlukan kelas anonim yang panjang, terutama dalam operasi koleksi dan Stream.

Sintaks dasar: (parameter) -> ekspresi. Contoh dalam aplikasi: .filter(t -> t.getKategori().getJenis() == JenisTransaksi.PENGELUARAN).

2.4 Stream API

Stream API (Java 8+) menyediakan cara fungsional untuk memproses koleksi data secara deklaratif. Stream mendukung operasi intermediate (filter, map, sorted) dan operasi terminal (collect, count, sum, average). Dalam aplikasi ini, Stream API digunakan secara ekstensif untuk menghitung total pengeluaran, mengelompokkan data per kategori, dan memfilter transaksi berdasarkan periode waktu.

2.5 RESTful API

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun layanan web. RESTful API menggunakan HTTP method standar: GET (membaca data), POST (membuat data baru), PUT (memperbarui data), dan DELETE (menghapus data). Response dikirimkan dalam format JSON.

2.6 Spring Boot

Spring Boot adalah framework Java yang mempercepat pengembangan aplikasi berbasis Spring dengan konfigurasi minimal (convention over configuration). Spring Boot menyediakan embedded server (Tomcat), auto-configuration, dan dependency management via Maven/Gradle, sehingga pengembang dapat fokus pada logika bisnis tanpa banyak konfigurasi boilerplate.

2.7 Spring Data JPA

Spring Data JPA adalah modul Spring yang menyederhanakan akses database menggunakan Java Persistence API (JPA). Dengan meng-extend interface JpaRepository, developer mendapatkan operasi CRUD secara otomatis tanpa perlu menulis query SQL secara manual. Spring Data JPA juga mendukung query method yang dibuat hanya berdasarkan penamaan method.

2.8 MySQL

MySQL adalah sistem manajemen basis data relasional (RDBMS) open-source yang banyak digunakan dalam aplikasi web. Dalam proyek ini, MySQL dijalankan menggunakan dua opsi: Docker Container untuk lingkungan yang terisolasi, dan XAMPP untuk penggunaan tanpa Docker. Spring Boot terhubung ke MySQL melalui JDBC dengan konfigurasi pada file application.properties.

4.2.2 Entity Kategori

Class Kategori merepresentasikan tabel kategori di database. Menggunakan anotasi JPA untuk pemetaan ORM (Object-Relational Mapping), dengan enkapsulasi data melalui getter dan setter. 

@Entity
@Table(name = "kategori")
public class Kategori {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String nama;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private JenisTransaksi jenis;

    public Kategori() {}

    public Kategori(String nama, JenisTransaksi jenis) {
        this.nama = nama;
        this.jenis = jenis;
    }

    // Getter dan Setter
    public Long getId()              { return id; }
    public void setId(Long id)       { this.id = id; }
    public String getNama()          { return nama; }
    public void setNama(String nama) { this.nama = nama; }
    public JenisTransaksi getJenis() { return jenis; }
    public void setJenis(JenisTransaksi jenis) { this.jenis = jenis; }
}

4.3.2 Map — Pengeluaran Per Kategori

// Collection Framework - Map: key=nama kategori, value=total jumlah
public Map<String, Double> getPengeluaranPerKategori(int bulan, int tahun) {
    return getByBulan(bulan, tahun).stream()
        .filter(t -> t.getKategori().getJenis() == JenisTransaksi.PENGELUARAN)
        .collect(Collectors.groupingBy(
            t -> t.getKategori().getNama(),
            Collectors.summingDouble(Transaksi::getJumlah)
        ));
}

4.3.3 Set — Tahun Unik

// Collection Framework - Set: tidak ada duplikasi tahun
public Set<Integer> getAllTahun() {
    return transaksiRepository.findAll().stream()
        .map(t -> t.getTanggal().getYear())
        .collect(Collectors.toSet());
}

4.4 Implementasi Lambda Expression dan Stream API

Lambda Expression dan Stream API digunakan secara ekstensif di layer Service untuk memproses data keuangan secara fungsional dan efisien.

4.4.1 Filter Transaksi Per Bulan (Stream + Lambda)

public List<Transaksi> getByBulan(int bulan, int tahun) {
    LocalDate awal  = YearMonth.of(tahun, bulan).atDay(1);
    LocalDate akhir = YearMonth.of(tahun, bulan).atEndOfMonth();
    return transaksiRepository.findByTanggalBetween(awal, akhir);
}

4.4.2 Hitung Total Pengeluaran (Stream + mapToDouble)

public Double getTotalPengeluaranBulan(int bulan, int tahun) {
    return getByBulan(bulan, tahun).stream()
        .filter(t -> t.getKategori().getJenis() == JenisTransaksi.PENGELUARAN) // Lambda
        .mapToDouble(Transaksi::getJumlah)   // Method reference (bentuk Lambda)
        .sum();                               // Terminal operation
}

4.4.3 Hitung Total Pemasukan (Stream + Lambda)

public Double getTotalPemasukanBulan(int bulan, int tahun) {
    return getByBulan(bulan, tahun).stream()
        .filter(t -> t.getKategori().getJenis() == JenisTransaksi.PEMASUKAN)
        .mapToDouble(Transaksi::getJumlah)
        .sum();
}

4.4.4 Ringkasan Per Bulan (Stream + Iterasi)

// Menghasilkan data 12 bulan untuk grafik bar
public List<Map<String, Object>> getRingkasanPerBulan(int tahun) {
    List<Map<String, Object>> hasil = new ArrayList<>();
    for (int b = 1; b <= 12; b++) {
        int bulanFinal = b;
        Map<String, Object> row = new HashMap<>();
        row.put("bulan",       b);
        row.put("pemasukan",   getTotalPemasukanBulan(bulanFinal, tahun));
        row.put("pengeluaran", getTotalPengeluaranBulan(bulanFinal, tahun));
        hasil.add(row);
    }
    return hasil;
}

4.4.5 Filter Kategori Pengeluaran (Stream di KategoriService)

// Stream API + Lambda di KategoriService
public List<Kategori> getPengeluaran() {
    return kategoriRepository.findAll().stream()
        .filter(k -> k.getJenis() == JenisTransaksi.PENGELUARAN)  // Lambda
        .toList();
}

4.5 Implementasi JPA Repository

Setiap entity memiliki interface Repository yang meng-extend JpaRepository. Spring Data JPA secara otomatis menyediakan implementasi CRUD tanpa perlu menulis SQL. 

// JPA Repository dengan query method otomatis
@Repository
public interface TransaksiRepository extends JpaRepository<Transaksi, Long> {

    // Spring Data JPA membuat query: SELECT * FROM transaksi
    // WHERE tanggal BETWEEN :start AND :end
    List<Transaksi> findByTanggalBetween(LocalDate start, LocalDate end);

    // SELECT * FROM transaksi WHERE kategori_id = :kategoriId
    List<Transaksi> findByKategoriId(Long kategoriId);

    // SELECT * FROM transaksi ORDER BY tanggal DESC
    List<Transaksi> findAllByOrderByTanggalDesc();
}

@Repository
public interface BudgetRepository extends JpaRepository<Budget, Long> {
    // Query method: cari budget berdasarkan bulan DAN tahun
    Optional<Budget> findByBulanAndTahun(Integer bulan, Integer tahun);
}

4.6 Implementasi RESTful API

Controller menggunakan anotasi @RestController dan @RequestMapping untuk mendefinisikan endpoint REST. Setiap method HTTP di-mapping ke operasi CRUD yang sesuai. 

@RestController
@RequestMapping("/api/transaksi")
public class TransaksiController {

    @Autowired
    private TransaksiService transaksiService;

    // GET - Read All
    @GetMapping
    public ResponseEntity<List<Transaksi>> getAll() {
        return ResponseEntity.ok(transaksiService.getAll());
    }

    // POST - Create
    @PostMapping
    public ResponseEntity<Transaksi> create(@RequestBody Transaksi transaksi) {
        return ResponseEntity.ok(transaksiService.save(transaksi));
    }

    // PUT - Update
    @PutMapping("/{id}")
    public ResponseEntity<Transaksi> update(
            @PathVariable Long id, @RequestBody Transaksi transaksi) {
        try {
            return ResponseEntity.ok(transaksiService.update(id, transaksi));
        } catch (RuntimeException e) {
            return ResponseEntity.notFound().build();
        }
    }

    // DELETE - Delete
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        transaksiService.delete(id);
        return ResponseEntity.ok().build();
    }

    // GET Stream API - Filter per bulan
    @GetMapping("/bulan/{bulan}/tahun/{tahun}")
    public ResponseEntity<List<Transaksi>> getByBulan(
            @PathVariable int bulan, @PathVariable int tahun) {
        return ResponseEntity.ok(transaksiService.getByBulan(bulan, tahun));
    }

    // GET Stream API - Pengeluaran per kategori (Map)
    @GetMapping("/per-kategori/bulan/{bulan}/tahun/{tahun}")
    public ResponseEntity<Map<String, Double>> getPerKategori(
            @PathVariable int bulan, @PathVariable int tahun) {
        return ResponseEntity.ok(
            transaksiService.getPengeluaranPerKategori(bulan, tahun));
    }

    // GET Collection Set - Tahun unik
    @GetMapping("/tahun")
    public ResponseEntity<Set<Integer>> getAllTahun() {
        return ResponseEntity.ok(transaksiService.getAllTahun());
    }
}

4.6.1 Dashboard Controller

@RestController
@RequestMapping("/api/dashboard")
public class DashboardController {

    @Autowired private TransaksiService transaksiService;
    @Autowired private BudgetService    budgetService;

    @GetMapping("/bulan/{bulan}/tahun/{tahun}")
    public ResponseEntity<Map<String, Object>> getDashboard(
            @PathVariable int bulan, @PathVariable int tahun) {

        double pengeluaran = transaksiService.getTotalPengeluaranBulan(bulan, tahun);
        double pemasukan   = transaksiService.getTotalPemasukanBulan(bulan, tahun);
        double sisa        = budgetService.getSisaBudget(bulan, tahun, pengeluaran);

        Map<String, Object> data = new HashMap<>();
        data.put("totalPengeluaran",      pengeluaran);
        data.put("totalPemasukan",        pemasukan);
        data.put("sisaBudget",            sisa);
        data.put("selisih",               pemasukan - pengeluaran);
        data.put("pengeluaranPerKategori",
            transaksiService.getPengeluaranPerKategori(bulan, tahun)); // Map
        data.put("ringkasanPerBulan",
            transaksiService.getRingkasanPerBulan(tahun));             // List
        return ResponseEntity.ok(data);
    }
}

// Fungsi API helper di app.js - integrasi front-end ke back-end
async function apiGet(url) {
    const res = await fetch(url);
    if (!res.ok) throw new Error('Gagal mengambil data');
    return res.json();
}

async function apiPost(url, data) {
    const res = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
    });
    return res.json();
}

4.8.1 Login Sederhana

Login diimplementasikan dengan endpoint POST /api/auth/login yang memvalidasi username dan password yang dikonfigurasi di application.properties. Setelah login berhasil, status tersimpan di localStorage browser. 

// application.properties
app.username=admin
app.password=admin123

// AuthController.java
@PostMapping("/login")
public ResponseEntity<Map<String, Object>> login(
        @RequestBody Map<String, String> body) {

    String username = body.get("username");
    String password = body.get("password");

    if (appUsername.equals(username) && appPassword.equals(password)) {
        return ResponseEntity.ok(Map.of(
            "success", true,
            "message", "Login berhasil",
            "username", username
        ));
    }
    return ResponseEntity.status(401).body(Map.of(
        "success", false,
        "message", "Username atau password salah"
    ));
}

Kesimpulan dan Saran

6.1 Kesimpulan

Berdasarkan hasil pengembangan dan pengujian aplikasi CatatKeu — Catatan Keuangan Pribadi, dapat disimpulkan bahwa:

  1. Aplikasi berhasil dibangun dengan arsitektur berlapis menggunakan Spring Boot sebagai framework back-end, dengan pemisahan yang jelas antara layer Entity, Repository, Service, dan Controller sesuai prinsip OOP.
  2. Collection Framework (List, Map, Set) berhasil diimplementasikan secara natural: List untuk menyimpan daftar data, Map untuk pengelompokan pengeluaran per kategori, dan Set untuk mengambil tahun unik dari data transaksi.
  3. Lambda Expression dan Stream API digunakan secara ekstensif untuk memproses data keuangan, menghitung total pengeluaran/pemasukan, dan menghasilkan data ringkasan bulanan secara fungsional dan efisien.
  4. RESTful API berhasil dibangun dengan 15 endpoint yang mencakup operasi CRUD untuk semua entitas serta endpoint analisis data menggunakan Stream API.
  5. Spring Data JPA menyederhanakan akses database dengan query method otomatis, menghilangkan kebutuhan penulisan SQL secara manual untuk operasi dasar.
  6. Integrasi front-end dan back-end berhasil dilakukan, di mana halaman HTML/JavaScript berkomunikasi dengan REST API menggunakan Fetch API dan merender data secara dinamis termasuk grafik interaktif berbasis Chart.js.
  7. Fitur poin plus berupa Login dan Dashboard Grafik berhasil diimplementasikan dengan tampilan yang profesional dan fungsional.

6.2 Saran

Untuk pengembangan lebih lanjut, beberapa saran yang dapat diberikan adalah:

  1. Autentikasi JWT — Mengimplementasikan JSON Web Token untuk keamanan autentikasi yang lebih robust dibandingkan sistem login sederhana saat ini.
  2. Multi-user — Menambahkan fitur registrasi dan manajemen pengguna sehingga aplikasi dapat digunakan oleh banyak pengguna secara terpisah.
  3. Export Laporan — Menambahkan fitur ekspor data ke format PDF atau Excel untuk keperluan pelaporan keuangan.
  4. Notifikasi — Menambahkan peringatan ketika pengeluaran mendekati atau melebihi budget yang telah ditetapkan.
  5. Deployment — Mendeploy aplikasi ke platform cloud (seperti Railway, Heroku, atau VPS) agar dapat diakses dari mana saja.
